Key idea
Binds inter-dependent buggy chunks into one "Buggy Block" via method (M) and field (F) dependency analysis, then fine-tunes CodeBERT on the unified representation
MRL
Minimized Repaired Locations — excludes blank/comment/null lines unless tagged FAULT_OF_OMISSION, reducing noise in the patch search space
Patch optimization
(1) Filter: removes duplicates, syntax errors, and unrelated patches  (2) Rank: rewards fewer edit actions using union denominator  (3) Combine: proportional top-k across locations
Bug taxonomy
T1 single-chunk · T2 single-chunk multi-location · T3 multi-chunk — system specifically targets T3 (hardest category)

03 Classification-Based Fault Localization

2025 – Present KCSE 2026
GitHub Paper (pp. 183–186)
PythonGPT-4oAST ParsingSWE-Bench Lite
74.6% file-path accuracy
52.3% symbol-level accuracy
$0.05 cost per issue
37.5% cheaper than Agentless
Classification
FULL_LOCATION — direct AST index lookup, no LLM (~$0.01) · PARTIAL — one LLM reasoning call on symbol summaries (~$0.04) · HINT — iterative agent search up to 5 rounds (~$0.05)
Repo indexing
Python ast module builds full symbol index per file (functions, class elements, module symbols); 3-map file resolver (basename/suffix/module path) — no embeddings needed
PARTIAL handler
Constructs symbol summaries (80-line excerpt + inferred call graph) → GPT-4o labels each EDIT_LOCATION / SYMPTOM_ONLY / NOT_RELATED
HINT handler
Agent iterates over search_class / search_method / search_code APIs backed by in-memory AST index; file scores decay by discovery iteration (earlier = higher weight)
Budget
Hard limits of $0.50/issue and 3,000 s; gracefully degrades by disabling optional LLM steps as limits approach
